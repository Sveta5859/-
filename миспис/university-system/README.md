 Описание предметной области

Модель описывает структуру университета:

- Университет (`University`) содержит факультеты (`Faculty`)
- Факультет включает кафедры (`Institute`) и управляется деканом (`AdministrativeEmployee`)
- Научные работники (`ResearchAssociate`) относятся к кафедрам и могут участвовать в проектах (`Project`) через сущность `Participation`
- Преподаватели (`Lecturer`) ведут курсы (`Course`)
- Административный персонал (`AdministrativeEmployee`) выполняет служебные операции, например, изменение e‑mail сотрудника (`Employee`)

Все эти сущности и связи отражены в Java‑коде и на UML‑диаграммах.

 Сценарии системы

Основные поддерживаемые сценарии (см. use case и sequence диаграммы):

 1. Управление факультетами и институтами деканом
- добавление и удаление кафедр на факультете
- хранение декана факультета
- добавление и удаление научных работников на кафедре
- создание проектов

 2. Работа преподавателя с курсами
- назначение курса (создание объекта `Course` и привязка к `Lecturer`)
- изменение количества часов курса
- отмена курса (удаление из списка курсов преподавателя)

 3. Работа научного работника с проектами
- участие в проекте с указанием количества часов (`ResearchAssociate.assignToProject`)

 4. Операции административного персонала
- изменение e‑mail произвольного сотрудника (`AdministrativeEmployee.changeEmail` через `Employee.updateEmail`)

 Запуск демонстрационной программы

1. Открыть папку с проектом в любой Java‑IDE или собрать проект из командной строки
2. Убедиться, что все Java файлы находятся в classpath
3. Запустить класс `Main`

 Компиляция и запуск из командной строки:

Скомпилировать все файлы
javac .java

 Запустить программу
java Main

Программа последовательно выполняет все основные сценарии и выводит результат в консоль:
- создание университета и факультета
- назначение декана и работа с институтами
- добавление/удаление научных работников
- участие научного работника в проекте
- изменение e‑mail сотрудника
- назначение, изменение и отмена курсов преподавателем
- итоговое состояние объектов системы
